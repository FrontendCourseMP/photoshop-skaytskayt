# Гистограмма и инструмент «Уровни» (Levels)

https://frontendcoursemp.github.io/photoshop-skaytskayt/lab3/

## Что внутри

- Загрузка PNG / JPG / GB7, сохранение в любой из этих форматов (включая GB7+маска).
- Диалог «Уровни…» в тулбаре открывает модальное окно (`<dialog>`) с гистограммой и слайдерами.
- Гистограмма строится по 256 бинам:
  - **Master** — по светлоте `0.2126·R + 0.7152·G + 0.0722·B` (та же формула, что и при экспорте в GB7).
  - **R / G / B / A** — по сырым значениям соответствующего канала.
  - Переключатель **Linear / Log** меняет ось ординат (`log = log1p(count) / log1p(max)`).
- Слайдер Input Levels: три маркера — точка чёрного, гамма, точка белого.
  - Чёрный и белый не пересекаются (минимальный зазор 1).
  - Гамма всегда лежит между ними; mapping положения в значение 0.1..9.9 экспоненциальный (`10^t`), поэтому центр = 1.0.
  - Числовые поля рядом дублируют ползунки для точного ввода.
- Параметры хранятся отдельно для Master и каждого из R / G / B / A — переключение канала не сбрасывает чужие настройки.
- LUT-таблицы:
  - `buildLut(black, white, gamma)` — линейный растяг + степенная гамма.
  - `applyComposed(channel, master)` — композиция «сначала per-channel, потом master».
  - `applyLuts(source, {r,g,b,a})` — пробег по `ImageData` с подстановкой через таблицы.
- Предпросмотр включается чекбоксом (по умолчанию активен). Снятие — мгновенный возврат к исходному изображению.
- Кнопки: **Сброс** обнуляет параметры всех каналов, **Отмена** закрывает диалог без правок, **Применить** записывает результат в текущее изображение.
- Производительность: пересчёт превью идёт через `requestAnimationFrame` (соседние изменения ползунков схлопываются в одно обновление за кадр), `ChannelLuts` мемоизированы через `useMemo`.

## Локальный запуск

```bash
npm install
npm run dev
```

Сборка под GitHub Pages — `npm run build`, артефакты в `dist/`.
